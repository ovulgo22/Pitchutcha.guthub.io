MS-DOS: O Motor da Revolução do PC

Enquanto o UNIX prosperava na academia com sua filosofia de design elegante, o mundo dos negócios precisava de algo que simplesmente funcionasse. O MS-DOS não era elegante, mas era pragmático, estava disponível e se tornou a base para o maior ecossistema de hardware da história.

A história do MS-DOS (Microsoft Disk Operating System) é menos sobre inovação técnica e mais sobre uma extraordinária visão de negócios. Ele se tornou o sistema operacional padrão para a computação pessoal nos anos 80 e início dos anos 90, não por ser o melhor, mas por estar no lugar certo, na hora certa, com o modelo de negócios certo.

As Origens: O Acordo com a IBM e o 86-DOS

Como vimos, quando a IBM precisava de um sistema operacional para seu PC em 1980, a Microsoft viu uma oportunidade. Em vez de construir um do zero, eles foram à Seattle Computer Products e licenciaram, e depois compraram, um sistema operacional chamado 86-DOS (também conhecido como QDOS - Quick and Dirty Operating System). O 86-DOS foi projetado por Tim Paterson para ser funcionalmente semelhante ao sistema operacional mais popular da época para microcomputadores, o CP/M, o que facilitaria a adaptação de programas existentes.

A Microsoft renomeou-o para PC-DOS para a IBM e, crucialmente, manteve os direitos de licenciá-lo para outros fabricantes como MS-DOS. Foi essa decisão que transformou a Microsoft de uma empresa de linguagens de programação em um gigante de sistemas operacionais.

Características Principais

Para um usuário moderno, o MS-DOS pareceria incrivelmente primitivo. Suas características definidoras foram:

  • Sistema Monotarefa e Monousuário: Apenas um programa podia ser executado de cada vez, e o sistema foi projetado para ser usado por apenas uma pessoa. Não havia conceito de contas de usuário ou permissões.
  • Interface de Linha de Comando (CLI): Toda a interação ocorria através de um prompt de comando, tipicamente o famoso C:\>. Os usuários precisavam digitar comandos como DIR para listar arquivos, COPY para copiar, e DEL para apagar.
  • Sistema de Arquivos FAT: Ele usava o sistema de arquivos FAT (File Allocation Table) para organizar os dados em disquetes e, mais tarde, em discos rígidos.
  • Gerenciamento de Memória Limitado: Uma de suas limitações mais notórias foi a barreira de 640 KB de memória convencional. Os programadores tinham que usar truques complexos (memória estendida e expandida) para acessar mais RAM, um desafio constante à medida que os aplicativos se tornavam mais sofisticados.

O Rei dos Clones

O sucesso esmagador do MS-DOS estava diretamente ligado ao sucesso do ecossistema de clones do IBM PC. Como qualquer empresa podia construir um hardware compatível com o PC, elas precisavam de um sistema operacional para rodar nele. A Microsoft estava mais do que feliz em vender-lhes o MS-DOS a um preço razoável.

Essa simbiose criou um ciclo virtuoso: mais clones de PC vendidos significavam mais licenças de MS-DOS, o que incentivava os desenvolvedores a escrever mais software para MS-DOS, o que, por sua vez, tornava a plataforma de clones de PC ainda mais atraente para os consumidores.

O Legado do DOS: A Fundação para o Windows

O MS-DOS introduziu milhões de pessoas à computação pessoal e criou o maior mercado de software que o mundo já tinha visto, com aplicativos como Lotus 1-2-3, WordPerfect e inúmeros jogos. Seu legado mais duradouro, no entanto, foi servir como a espinha dorsal para as primeiras versões do Microsoft Windows.

O Windows 1.0, 2.0, 3.x, 95, 98 e Me não eram sistemas operacionais independentes no sentido moderno. Eram, na verdade, ambientes operacionais gráficos que rodavam em cima do MS-DOS. O computador ainda inicializava o DOS primeiro, e o Windows era então carregado. Somente com o Windows NT e, eventualmente, o Windows XP, a Microsoft fez a transição completa para um sistema operacional que não dependia mais da base do MS-DOS.
