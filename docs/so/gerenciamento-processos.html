Gerenciamento de Processos

Como o Sistema Operacional cria a ilusão de que múltiplos programas estão rodando simultaneamente em uma única CPU.

Quando você abre um navegador, um editor de texto e um player de música ao mesmo tempo, cada um desses programas roda como um processo. O gerenciamento de processos é a tarefa do SO de administrar a criação, execução e finalização desses processos, distribuindo o tempo da CPU entre eles de forma justa e eficiente.

O Que é um Processo?

Um processo é mais do que apenas o código de um programa. É uma instância de um programa em execução, contendo seu estado atual e recursos alocados. O SO armazena todas as informações sobre um processo em uma estrutura de dados chamada Bloco de Controle de Processo (PCB). O PCB inclui:

  • O estado atual do processo (pronto, executando, etc.).
  • O valor dos registradores da CPU.
  • Informações sobre a memória alocada.
  • Uma lista de arquivos abertos e outros recursos.

Os Estados de um Processo

Durante sua vida, um processo passa por vários estados:

  1. Novo (New): O processo está sendo criado.
  2. Pronto (Ready): O processo está na memória e pronto para ser executado, aguardando sua vez de usar a CPU.
  3. Executando (Running): O processo está atualmente utilizando a CPU para executar suas instruções.
  4. Bloqueado (Blocked/Waiting): O processo está esperando por um evento externo, como o término de uma operação de leitura do disco ou a chegada de dados da rede. Ele não pode usar a CPU mesmo que ela esteja livre.
  5. Terminado (Terminated): O processo concluiu sua execução e seus recursos estão sendo liberados pelo SO.

O Escalonador e a Troca de Contexto

O componente do SO responsável por decidir qual processo no estado "Pronto" deve receber a CPU é chamado de escalonador (scheduler). Em sistemas multitarefa, o escalonador alterna a CPU entre os processos centenas de vezes por segundo.

Essa ação de alternar a CPU de um processo para outro é chamada de troca de contexto (context switch). Para isso, o SO deve:

  1. Salvar o contexto do processo A (todos os valores dos registradores da CPU) em seu PCB.
  2. Carregar o contexto do processo B a partir do PCB dele para os registradores da CPU.
  3. Continuar a execução a partir do ponto em que o processo B parou.

A Ilusão da Simultaneidade: A troca de contexto acontece tão rápido que, para um usuário humano, parece que todos os programas estão rodando ao mesmo tempo. Na realidade, em uma CPU de núcleo único, apenas um processo está verdadeiramente em execução em um determinado instante no tempo. Este conceito é chamado de concorrência.

Embora a troca de contexto seja essencial para a multitarefa, ela tem um custo (overhead), pois nenhum trabalho útil é realizado durante a troca. O design de um bom escalonador busca maximizar o uso da CPU e minimizar a latência, proporcionando uma experiência de usuário fluida e responsiva.
